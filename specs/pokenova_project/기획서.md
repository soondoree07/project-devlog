## 1. 프로젝트 개요

포케노바는 **포켓몬 챔피언스** 및 일반 포켓몬 유저를 위한 **도감 + 퀴즈 플랫폼**이다.  
포켓몬 맞추기 퀴즈(5종), 오늘의 포켓몬 뽑기, 포켓몬/기술/도구 도감을 제공한다.

### 핵심 가치
- 포켓몬 챔피언스 데이터 중심 — `in_champions` 필터로 챔피언스 전용 뷰 지원
- pochams 공유 DB (Neon PostgreSQL) 경유로 데이터 최신화 — 별도 데이터 관리 불필요
- 바닐라 HTML/JS + Vanilla CSS 구조 — 빌드 없이 GitHub Pages·Vercel 즉시 배포

## 2. pochams_project와의 관계

### 2-1. 데이터 흐름

```
pokenova_project/data/          pochams_project (DB)             pokenova (브라우저)
─────────────────────────       ──────────────────────────       ──────────────────────
quiz_pokemon.json       ──seed──▶  pokemon_data 테이블  ──API──▶  포켓몬 데이터
quiz_moves.json         ──seed──▶  move_data 테이블     ──API──▶  기술 데이터
quiz_items.json         ──seed──▶  item_data 테이블     ──API──▶  도구 데이터
quiz_abilities.json     ──seed──▶  ability_data 테이블  ──API──▶  특성 데이터
learnsets.json          ──seed──▶  pokemon_learnset      ──API──▶  배우는 기술
```

- **pokenova가 원천 데이터 관리자.** JSON 파일을 수정하고 pochams `npm run seed`(또는 `npm run upsert`)로 DB 반영한다.
- 기술·도구·특성은 브라우저가 pochams API(`https://pochams.com/api/data/...`)를 직접 fetch(localStorage 6시간 캐싱).

> **⚠️ 2026-07-09 변경 — 포켓몬/배우는기술은 pochams API 미사용.**
> pochams 독립 개편으로 pochams API가 챔피언 205마리만 반환하게 되면서, 포케노바가 **자체 마스터 데이터**로 전환했다.
> - `data/pokedex.json` — 전국도감 1025 + 폼 + `inChampions` on/off 플래그. 포케노바가 `inChampions` 단일 원천(마스터). `scripts/build_pokedex.js`로 (재)생성.
> - `data/learnsets.json`(1285) — 배우는기술 전체. movedex·entry가 로컬에서 직접 로드.
> - 포켓몬을 쓰는 5개 페이지(pokedex/index/today/movedex/entry) 모두 로컬 `data/pokedex.json`을 fetch.
> - 포챔스 추가/제거 = `pokedex.json`에서 `inChampions`를 on/off → `cd pochams_project && npm run sync`.
>   브릿지(`champions_from_pokedex.py`)가 `champions_pokemon_ids.json`을 생성하고, 위키 기술폭 수집·소스 생성·DB seed까지 자동 실행된다.

## 3. 주요 기능

### 3-1. 포켓몬 맞추기 퀴즈 (index.html)

**퀴즈 5종:**

| 탭 | 설명 | 힌트 방식 |
|---|---|---|
| 포켓몬 (b) | 실루엣·타입 힌트 → 이름 맞추기 | 실루엣 공개 → 타입 공개 |
| 기술 (m) | 설명 공개 → 기술 이름 맞추기 | 설명 점진 공개 |
| 특성 (a) | 설명 공개 → 특성 이름 맞추기 | 설명 점진 공개 |
| 도구 (i) | 설명 공개 → 도구 이름 맞추기 | 설명 점진 공개 |
| 능력치 (st) | HP/공/방/특공/특방/스피드 6종 + 타입 공개 → 이름 맞추기 | 복수 정답 허용 |

**공통 게임 규칙:**
- 목숨 5개 (능력치 퀴즈: 10개), 모두 소진 시 게임 오버
- 정답 5의 배수마다 보상: 목숨 부족 시 +1, 꽉 찼으면 힌트 +1
- 세대 필터: 1~9세대 복수 선택 가능 (전체 포함) — 리전폼/메가/거다이맥스 세대 올바르게 분류
- 세대 한정 모드: 게임 오버 후 리더보드 등록 불가
- 포챔스 필터: 챔피언스 등장 포켓몬만 출제
- **퀴즈 5종 전체 정답 시 '완전 정복!' 축하 오버레이** — 리더보드 등록 가능 모드면 등록 UI 연결

**리더보드:**
- Firebase Firestore 기반
- 5종 퀴즈 각각 별도 리더보드
- 관리자 페이지(admin.html)에서 모드별 조회 및 초기화 가능

### 3-2. 오늘의 포켓몬 (today.html)

- 몬스터볼 클릭 → `videos/pokeball-open.mp4` 영상 재생 → 랜덤 포켓몬 공개
- 클릭 시 즉시 스킵하여 포켓몬 공개 (영상 재생 중 클릭)
- 모바일: 클릭 전 바운스 포켓볼 플레이스홀더, 클릭 시 영상 노출 (검정화면 방지)
- 공개된 포켓몬: 특성(랜덤 1개, 뽑는 순간 localStorage 저장) + 능력치 6종 표시
- **하루 1회 뽑기 제한** + 컬렉션 중복 제외 + 보관하기/놓아주기/다시뽑기 기능
- **컬렉션 100마리 제한** — 초과 시 교체할 포켓몬 직접 선택 스왑 모달
- 컬렉션 localStorage 저장, 그리드 표시, 칩 클릭 시 상세 모달

### 3-3. 포켓몬 도감 (pokedex.html)

- 포켓몬 전체 목록 카드 그리드 (스프라이트, 이름, 타입, 번호)
- **포챔스 필터**: 챔피언스 등장 포켓몬만 표시 (inChampions=true)
- 세대 필터, 타입 필터
- **알트폼 표시**: 원종 바로 옆에 알트폼 카드 인터리브 — 같은 도감 번호로 표시
  - 폼 라벨: 루가루암 '한낮의 모습', 스트린더 '하이한 모습', 우라오스 '일격의 태세' 등 원종 폼 라벨 표시
  - 히스이 리전 폼: '히스이' 뱃지 표시 (신비록·사마자르·다투곰·포푸니크·장침바루·러브로스 등)
  - 대쓰여너 수컷/암컷: 오른쪽 위 뱃지 표시
- 메가진화: canonical_mega.json 기반 연결 (포챔스 ZA 메가 5종 포함)
- 클릭 시 통계 팝업 또는 포켓몬 상세(entry.html) 연결
- **id < 100000 필터**: alt폼 DB 별도 행 중복 렌더링 방지

### 3-4. 포켓몬 상세 (entry.html)

- 포켓몬 스프라이트, 이름, 번호, 타입, 종족값
- **배우는 기술 목록**: pochams `/api/data/learnsets` (DB 기반) — 레벨업/머신/교배 구분
- 메가진화 연결 (rayquaza_mega, dragon_ascent 등 특수 케이스 처리)
- en 필드 underscore 정규화 기반 룩업

### 3-5. 기술 도감 (movedex.html)

- 기술 전체 목록, 타입/분류 필터
- **포챔스 필터**: in_champions=true 기술만 표시
- 기술 클릭 시 설명, 위력, 명중, PP, 분류 표시
- 배우는 포켓몬 목록 (learnsets 기반)
- en 필드 underscore 정규화 기반 기술 조회

### 3-6. 도구 도감 (itemdex.html)

- 도구 전체 목록
- **포챔스 필터**: 챔피언스 등록 도구 117종만 표시
- 카테고리 필터 (메가스톤 게임 등장 순서 정렬: XY→ORAS→커스텀, 동일 세대 내 가나다순)
- 도구 설명: 포켓몬 위키 SV/최신버전 기준 (491/659개, 기술머신/열매류 제외)
- 특수볼/아프리콘볼 분류 별도 처리

- **기본폼 (base form)**: 27마리에 실드폼/수컷/노말폼 등 기본 forms 항목 삽입 (type=base)
