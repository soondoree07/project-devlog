## 1. 한 줄 요약
메이플스토리 본진 파티의 주간/월간 보스 클리어 + 전리품 분배 내역을 기록·조회하는 1인용 정적 웹사이트. Vanilla HTML + ES Module + localStorage. 빌드 시스템 없음.

## 2. 기술 스택

## 3. 핵심 기능

### 3.1 파티 관리
- 새 파티 생성 (이름 + 닉네임 6칸 그리드, Enter로 다음 칸 이동, 빈 칸은 무시, 1명 이상 입력 시 저장)
- 파티 목록 조회 / 카드 클릭으로 파티 상세 진입
- **파티 카드 우상단 × 버튼**으로 삭제 (호버 시 노출, 모바일은 옅게 항상 노출, confirm으로 기록 N건도 함께 삭제 안내)
- 파티 상세 우상단 "삭제" 버튼도 동일 동작
- **파티원 추가** (v0.4) — 두 곳에서 가능:
  - 메인 파티 카드 멤버 그리드 끝 "+ 추가" 칩 (카드 클릭 이동은 stopPropagation으로 막음)
  - 파티 상세 파티원 strip 끝 "+ 추가" 칩
  - 클릭 시 모달로 닉네임 input → trim/중복 검사 후 `updateParty`로 members 배열에 append

### 3.2 캘린더 뷰
- 월 단위 그리드, **메이플 주차 시작 (목→수)** — 요일 헤더: 목/금/토/일/월/화/수
- 토·일은 빨간색 강조 (`--accent-rose`)
- 오늘 날짜 골드 강조
- ←/→ 버튼으로 월 이동 + **모달 닫고 재렌더해도 보고 있던 월 보존** (`calendarViewByParty` Map)
- 셀 클릭 → 그날 모달
- 셀에 보스별 색상 pill (최대 4개 + `+N` 표시)

### 3.3 진행도 위젯
- **이번 주 현황** — 주간 7보스 chip, 클리어한 보스만 색상 활성화 (`color-mix`)
- **이번 주 1인 분배액** — 결정석 합 ÷ 파티원 수, 옆에 결정석 이미지(`png/결정석.webp`) 클릭 시 `#/crystals`로 이동
- **이번 달 현황** — 검마 chip + 1인 분배액 + 월간 결정석 이미지(`png/월간보스결정석.webp`)
- 분배액 합산은 **`cycle === 'weekly'` / `cycle === 'monthly'`만**으로 가드 (월간 검마가 같은 주에 잡혀도 주간 합에 안 더해짐)

### 3.4 회차 기록 흐름

**날짜 모달 (캘린더 셀 클릭)**
- 채널 picker 버튼 그리드 (1 / 20세이상 / 2 / 3 / ... / 39, 8열, 드롭다운 X)
- 회차가 1건이라도 있으면 그 채널로 잠김
- 회차 카드 N개 (보스 / 채널 / 상자 연 사람 / 기본 보상 / **참여자 N인** / 전리품 목록 + 삭제 ×)
- "+ 보스 기록 추가" 버튼 — 채널 선택 전엔 비활성화

**기록 추가 폼**
- 보스 select — **같은 주 클리어한 주간 보스 / 같은 달 클리어한 검마는 자동 제외**
- **이번 회차 참여자** 토글 그리드 (v0.4 신규) — 파티원 중 이번에 간 사람만 토글. 후보는 파티 생성 시 등록된 멤버로 한정. **같은 날 직전 회차의 멤버를 자동 미리 채움 / 직전 회차가 없으면 파티 전체.** 회차 참여자 변경 시 상자 연 사람·기본 보상·전리품 taker의 select 옵션도 그 멤버로만 좁아짐.
- 상자 연 사람 select / 기본 보상 select (form-row 2열) — 직전 회차의 값으로 자동 미리 채움
- 전리품 입력은 **타일 그리드** (이미지 + 이름, 토글 선택). select 폐기.
- 선택된 전리품마다 [이름 라벨] [먹은 사람] [가격] **[분배 체크박스]** [×] 행으로 자동 추가
  - **분배 ON (디폴트):** 가격을 회차 참여자 N명에게 1/N씩 분배. taker select는 비활성 + '분배: 전원 N등분' 표기.
  - **분배 OFF:** taker 1명에게 가격 전액. taker 필수.
- **저장 필수: 보스 + 회차 참여자 1명 이상 + 상자 연 사람.** 기본 보상 / 전리품은 비워도 OK.

### 3.5 결정석 가격 편집 페이지 (`#/crystals`)
- 진행도 위젯의 결정석 이미지 클릭 시 진입
- 주간 보스 8종 (유피테르 포함) → 가로 2열 카드
- 월간 보스 검마 → 별도 섹션 카드
- 카드 좌측 보더 = 보스 색
- 페이지 하단 가운데 저장 버튼 1개 (한 번에 모든 가격 저장 → `crystalOverrides` 맵에 보관)
- 빈/0/숫자 아닌 값은 자동으로 기본값(BOSSES.crystal)으로 복귀

### 3.6 백업/복원
- `↓ 백업` 버튼 — 전체 데이터 JSON 다운로드 (`maple-boss-backup-YYYY-MM-DD.json`)
- `↑ 복원` 버튼 — JSON 업로드, confirm 후 덮어쓰기
- `crystalOverrides` / `reservations`도 백업/복원에 포함

### 3.7 미래 날짜 예약
- 캘린더에서 **미래 날짜 클릭** → 예약 모달
- `<input type="time">`으로 시간 입력 → "+ 추가"
- **하루에 1예약**만 (1건이라도 있으면 입력 폼 숨김 / 카드 + × 삭제만 노출)
- 캘린더 셀에는 골드 점선 보더 pill로 시간(`HH:MM`) 표시. 셀에 `has-reservation` 클래스 부여

### 3.8 월별 누적 사이드바
- 파티 상세 우측 sticky 320px 컬럼
- 오늘 기준 **이번 달 / 1달 전 / 2달 전** 카드 3개
- 헤더: `YYYY년 M월` + 우측에 **인당 총 결정석** (그 달 weekly+monthly 결정석 합 ÷ 인원, 0이면 `0억`, override 반영)
- 본문: 회차 전리품을 **최근 날짜순**으로 평탄화 → `[이미지][이름(유니크 도미넌트 컬러)][나누기 전 가격]` 행
- 회차 0건이면 "기록 없음"

### 3.9 채널 룰렛 사이드 카드
- 파티 상세 좌측 sticky 220px 컬럼 (룰렛 + 사다리타기 wrapper `side-left`)
- 슬롯 윈도우 + 시안/보라 그라데이션 "🎰 뽑기" 버튼
- 버튼 누르면 1.1초 동안 굴러가며 채널 후보를 스피닝 → eased tick interval(35ms→180ms) → CHANNELS 1개로 멈춤
- `roulette-shake` (스피닝 중) / `roulette-pop` (착지 시) 키프레임 애니메이션

### 3.10 파티원별 주간 수익 (v0.4 신규)
- 캘린더 바로 밑에 카드 형태로 노출
- 메이플 주차(목 0시 ~ 수 23:59) 기준 멤버별 수익을 합산
- 각 멤버의 수익 = Σ(그 주에 본인이 회차 참여자에 포함된 회차들에 대해)
  - **결정석:** `getEffectiveCrystal(boss) / 회차 인원수`
  - **분배 전리품:** `lt.price / 회차 인원수` (회차 참여자 1몫)
  - **단독 전리품:** `lt.price` 전액 (taker가 본인일 때만)
- 총수익 desc 정렬, aqua→purple 그라디언트 progress bar로 비중 표시
- 이번 주 회차 0건이면 "이번 주 회차 기록이 없어요"

### 3.11 사다리타기 카드 (v0.4 신규)
- 룰렛 카드 바로 아래 (좌측 사이드 컬럼)
- **2단계 UX:**
  1. **입력 단계 (가림)** — 인원수 input(2 ~ 파티 총원, 디폴트 2) + 이름 input N개. SVG 사다리는 블러 처리 + 🎴 cover로 가려짐. "🎲 뽑기" 버튼.
  2. **공개 단계** — 뽑기 누르면 새 사다리(랜덤 가로줄) + 결과(O 1명, X N-1명) 배치 + 공개. 이름 input들이 **클릭 가능한 색상 칩**으로 전환.
- **이름 칩 클릭** — 그 사람의 path를 색상 stroke로 그려가며(stroke-dashoffset 애니메이션) 점이 따라가는 모습이 보인다(1.5초, cubic ease). 도착 컬럼의 결과 셀이 outline + pop으로 강조.
- 다른 이름 클릭 시 이전 trace 제거 후 새로 그림
- 칩 색: aqua / purple / rose / gold / green / red 순환
- 카드 하단에 🏆 당첨자 안내 + "👆 이름을 눌러서 사다리를 따라가보세요" 힌트
- "🎲 다시 뽑기"로 입력 단계로 복귀
- 1인 파티는 "2명 이상부터 가능해요" 안내

### 3.12 회차 기록 수정 (v0.4 신규)
- 날짜 모달의 회차 카드에 ✎ 수정 버튼 (× 옆)
- 클릭 시 기록 추가 폼 재사용. 모든 필드(보스/회차 참여자/상자 연 사람/기본 보상/전리품 + taker + 가격 + 분배) prefill
- 보스 select은 같은 주/달 클리어 보스를 가리되 **자기 회차의 보스는 옵션에 남김** (그대로 유지하거나 다른 보스로 바꾸기 위해)
- 보스를 바꾸면 lootEntries 비우는 가드. 같은 보스로 두면 prefill 그대로 유지
- 저장은 `Storage.updateRun(existingRun.id, payload)` — 같은 회차 ID 유지

## 4. 데이터 모델

```js
{
  parties: [
    { id, name, members: [닉네임...], createdAt }
  ],
  bossRuns: [
    {
      id, partyId, date,
      boss,                          // 'seren' | ... | 'blackmage'
      channel,                       // "1" | "20세이상" | "2" ... "39"
      opener,                        // 상자 연 사람 (필수)
      baseReward,                    // 기본 보상 받은 사람 (선택)
      memberSnapshot: [닉네임...],   // v0.4부터 회차 실제 참여자(파티 일부 가능)
      loot: [{ item, taker, price, shared }]
        // price 단위: 억 (number | null)
        // shared: true=회차 N명 분배 / false=taker 전액. 미정의(legacy)는 단독 해석.
    }
  ],
  reservations: [                    // 미래 날짜 보스 일정 (하루 1건)
    { id, partyId, date, time, createdAt }   // time: "HH:MM"
  ],
  crystalOverrides: {                // 사용자가 수정한 보스별 결정석 가격
    [bossId]: number(억)
  }
}
```

## 7.2 디자인 / 테마
- **배경:** `background/`의 6장(리버스시티/별이삼켜지는심해/빛이마지막으로닿는곳/산호숲가는길/생명의동굴/정령의나무가있는곳) 중 페이지 로드마다 랜덤 1장. `html`에 fixed cover로 깔고 어두운 그라디언트 overlay(0.18~0.40)
- **팔레트:** 시안/aqua + 보라
  - `--accent-gold #4ee5f5` (구 골드 자리. CSS 토큰명만 유지)
  - `--accent-purple #a78bff`, `--accent-rose #ff7eb6`
  - `--accent-aqua #4ee5f5` (룰렛 등 별칭)
- **카드:** `rgba` 반투명 + `backdrop-filter: blur(10px) saturate(125%)`로 사진 위 가독성

> **CSS variable url() 함정:** variable의 `url()`은 *그 variable이 사용되는 stylesheet 위치* 기준으로 풀림. `/css/style.css`에서 `var(--bg-image)`를 쓰면 `css/background/...`로 잘못 풀려 404. **CSS 디폴트도 JS override도 모두 `../background/` prefix 통일.**
